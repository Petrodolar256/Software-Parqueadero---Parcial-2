# Software Parqueaderos - Página Web

Petrodolar256 te brinda el mejor software en formato de página web para administrar tu parqueadero.
Este README brindará una breve introducción de cómo funciona a nivel de desarrollo y de experiencia del usuario el portal web.

## Tecnologías usadas

**Nuestra tienda hace uso de tecnologías como HTML5, CSS3, JavaScript, Bootstrap, GitHub.**  


## Lo que logras usando este software: 

Al implementar este sistema, podrás gestionar de manera eficiente toda la información relacionada con los vehículos que ingresan y salen del parqueadero, optimizando el control y reduciendo errores manuales. Tendrás la capacidad de registrar entradas de forma rápida mediante un formulario estructurado que organiza los datos clave, como la placa, horarios, ubicación y tipo de vehículo. Esto permitirá mantener un historial claro y accesible en todo momento. Además, facilitará la actualización de la información en tiempo real, especialmente en lo relacionado con la salida de los vehículos y la reasignación de espacios. El sistema también mejora la organización interna al definir roles específicos de uso, garantizando que cada usuario interactúe únicamente con las funciones necesarias. En conjunto, lograrás mayor control operativo, mejor trazabilidad de los movimientos y una administración más ordenada y confiable del parqueadero.

## Sobre este repositorio

> Este repositorio hace parte del proceso de evaluación del parcial del primer corte de la asignatura de Programación Web de la Universidad Surcolombiana y cumple los requisitos:
>
- La aplicación deberá contar con un formulario de inicio de sesión personalizado, en el que el usuario
ingresará sus credenciales. Según el rol asignado, se mostrarán las vistas correspondientes al perfil
del usuario. Si el usuario no cuenta con los permisos requeridos para realizar alguna acción, se
mostrará una vista informativa personalizada indicando la restricción.
- Todos los servicios web (WS) de la aplicación deben estar documentados en Swagger, con
explicaciones detalladas y pruebas accesibles a través de la interfaz de Swagger UI.
- El código fuente debe ser subido a un repositorio en GitHub, junto con los scripts y/o queries para
la creación y alimentación de la base de datos.
